# DANGA UI 개선 스프린트 (결제/과금은 보류)

## 목표: “기능은 그대로” 두고, 첫인상/완성도/일관성/카피를 끌어올린다

---

## 0) 원칙(절대 규칙)

- 기능 로직/DB/RLS는 건드리지 않는다 (UI/카피/레이아웃만)
- 디자인은 “미니멀 + 경쾌(단번)” 톤으로 통일
- 모든 화면에 공통: 타이포/간격/버튼/카드/상태(로딩/빈상태/에러) 일관성 확보
- 모바일 우선(터치/간격/글자 크기)

---

## 1) UI 개선 범위(우선순위)

### P0 (바로 체감)

1. Header/Nav + Footer 통일 (모든 페이지)
2. Feed 카드(PostCard) 리디자인 (썸네일/정보/액션 정리)
3. Detail(/p/[id]) 레이아웃 정리 (이미지/액션/댓글 영역)
4. Form UI (/post/new) 개선 (업로드 단계/진행 상태/에러)
5. Empty/Loading/Error 상태 컴포넌트 공통화

### P1 (브랜딩/카피)

6. 랜딩 카피/섹션 문구 전면 정리
7. 버튼/토스트/모달 문구 톤 통일 (“단번에” 느낌)
8. 배지/랭킹 카피 다듬기 (Top 1%, Top 10% 표현)

### P2 (마감 디테일)

9. 아이콘/hover/transition/모서리/그림자 통일
10. 다크모드(선택)

---

## 2) 카피(멘트) 톤 가이드

### 2.1 톤

- 짧고 직관적: “단번에”, “바로”, “한 번에”
- 과한 밈/유행어는 최소, 대신 캐주얼한 확신 톤

### 2.2 공통 버튼/문구 추천

- 시작하기 → **로그인하고 시작**
- 구경하기 → **피드 보기**
- 업로드 → **코디 올리기**
- 좋아요 → **나이스!**
- 댓글 placeholder → **한 줄 피드백 남겨줘**
- 신고 → **신고하기**(짧게), 안내문은 친절하게
- 빈 상태:
  - 피드: “아직 코디가 없어요. 첫 번째로 올려볼래?”
  - 내 글: “아직 올린 코디가 없어요. 단번에 올리러 가자.”

## 3) 디자인 시스템(작게라도 도입)

### 3.1 기본 토큰(추천)

- radius: 16px (카드), 12px (버튼)
- shadow: 은은한 1~2단 (과하지 않게)
- spacing: 8px 단위
- typography:
  - H1: 36/40
  - H2: 24/28
  - body: 15~16
  - caption: 12~13

### 3.2 공통 컴포넌트(필수)

- <Button /> variants: primary/secondary/ghost/destructive
- <Card />
- <Badge />
- <Modal />
- <Toast /> (또는 inline alert)
- <EmptyState />
- <Skeleton /> (피드/상세)

> shadcn/ui를 쓰면 빠르게 정리 가능(이미 쓰고 있으면 그대로)

## 4) 화면별 개선 스펙(에이전트가 바로 적용 가능)

### 4.1 Header

- 로고(DANGA) 왼쪽 고정
- 가운데: Feed / Rank / Search(아이콘)
- 오른쪽:
  - 비로그인: “로그인하고 시작”
  - 로그인: 프로필 버튼 + 드롭다운(내 페이지/코디 올리기/로그아웃)
- 스크롤 시 sticky + 약한 blur(선택)

### 4.2 Feed 카드(PostCard)

- 이미지 비율 고정(1:1 또는 4:5 중 택1) → “그리드가 안정감”
- 카드 하단:
  - caption 1줄(ellipsis)
  - tags chips
  - 액션 바: 좋아요(하트) + 카운트, 댓글 카운트, 더보기
- hover: 살짝 lift (모바일에선 없음)
- “Top 1%/10%” 배지가 있으면 이미지 위 좌상단 pill로

### 4.3 Detail (/p/[id])

- 상단: 작성자(아바타/닉네임) + 작성일
- 가운데: 이미지(크게)
- 아래:
  - 좋아요/댓글/신고 버튼(정렬)
  - caption + tags
  - 댓글 영역:
    - 입력창 고정(상단/하단 중 택1)
    - 리스트는 깔끔한 타이포/간격

### 4.4 Post New (/post/new)

- 단계형 UI:
  1. 이미지 선택
  2. 미리보기 + 캡션/태그
  3. 게시하기
- 업로드 중/저장 중 상태를 명확하게
- 에러 문구는 “왜 안 되는지” 한 줄로

### 4.5 Admin

- 운영툴 느낌: 좌측 사이드바 + 표(리스트) + 디테일 패널
- 상태 배지(open/reviewing/resolved/rejected) 색 일관
