## 2. SEO 기본 세팅(필수)

### 2.1 메타/OG(공유 썸네일)

- `app/layout.tsx` 또는 각 route metadata 설정
- 최소:
  - title/description
  - openGraph(title, description, url, images)
  - twitter card

**권장 기본값**

- title: `DANGA — 단번에 가자`
- description: `코디 올리고, 투표로 단번에 평가받자.`

### 2.2 robots.txt / sitemap.xml

- Next.js에서 route handler로 제공
- robots:
  - 기본 허용
  - /admin 차단
- sitemap:
  - MVP는 정적(홈, feed, rank, me)
  - 게시글 동적 sitemap은 추후(Part 13.5)

### 2.3 canonical URL

- `https://danga.site` 기준으로 canonical 설정

## 3. 성능 최적화(필수)

### 3.1 이미지 최적화(가장 중요)

- `next/image` 사용
- image_url이 외부 도메인이면 `next.config.js`에서 허용 도메인 추가 필요
  - `img.danga.site`
  - 또는 R2 임시 URL 도메인(현재 우회 시)
- 썸네일: feed에서는 작은 크기로
- lazy loading 기본 적용

### 3.2 캐싱/페이지 전략

- feed/rank는 데이터가 자주 변하므로:
  - 기본은 client fetch
  - 또는 revalidate 전략(선택)
- 정적 페이지(landing)는 최대한 정적으로

### 3.3 번들/콘솔

- 프로덕션에서 과한 console.log 제거
- 필요하면 Sentry(선택)로 전환

## 4. 모바일 QA(필수)

### 4.1 체크 디바이스

- iPhone Safari
- Android Chrome
- 데스크탑 Chrome/Safari

### 4.2 체크 항목

- 랜딩: CTA가 화면에 잘 보임
- 로그인: 모달/리다이렉트 정상
- 업로드: 파일 선택/업로드 정상
- 피드: 스크롤/그리드 깨짐 없음
- 상세: 좋아요/댓글/신고 UI 터치 가능
- /admin: 일반 유저 접근 차단 정상

## 5. 운영 점검(필수)

### 5.1 에러/모니터링

- 최소:
  - Vercel Logs 확인
  - Supabase Logs 확인
- 권장:
  - Sentry 도입(선택)
  - 중요한 API route 에러에 대한 알림(후순위)

### 5.2 DB/RLS 점검

- 일반 유저로:
  - admin 데이터 접근 불가
  - 타인 posts/comments 수정/삭제 불가
- admin으로:
  - reports 조회/처리 가능
  - 대상 soft delete 가능

### 5.3 백업(선택)

- Supabase 백업 정책 확인
- 최소: 주기적으로 SQL dump/스냅샷 계획
